Challenge

In this lab, my partner and I learned to pseudocode. We first learned how pseudocoding works by using the method to describe how to make a sandwich. Then, we thought of a simply task that a computer could do. For our project we decided to pseudocode a basic program that would have a user fill in a Mad Lib.

Problems

By working together, my partner and I faced few problems. The hardest part of this lab was coming up with a task that was simple to pseudocode. We didn't want to get too lost in the details, so we tried to think of only the biggest steps in our program. Other than that, though, we managed to complete this lab easily.

Results

First, my partner and I pseudocoded a basic task- making a sandwich:

-Gather ingredients
-Bread, meat, cheese, veggies, and sauce
-Lay out two bread slices
-Put sauce on both bread slices
-Layer meat, cheese, and veggies on one of the bread slices
-Put bread slice on top of the meat, cheese, and veggies
-Cut sandwich
-Eat sandwich

Next, we psuedocoded a basic computer task- creating a madlib.

-Set up story template with empty variables representing ‘blanks’
-Have the user input words for the madlib
-Computer would ask for diff word types (verb, noun, ect)
-Assign user input to the empty variables
-Print final story with variables replaced with user input words
